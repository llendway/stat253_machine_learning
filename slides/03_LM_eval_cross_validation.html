Recall the model we fit using only ten car observations. The adjusted \(R^2\) is 1 and the RMSE is virtually zero. The model seems great.

As it turns out, we can often get good performance on the training dataset, especially if we make a complex enough model. This is called overfitting. With smaller datasets it is fairly easy to do this. But even with large datasets, we can also create and add enough variables with enough complexity to overfit the model.

In machine learning, we are mostly concerned about predicting future, unknown values. So, we want to have an honest assessment of how well our model performs on data it hasn’t seen. How do we do that?

I know! How about using that test dataset we held out initially?

Image Credit: https://giphy.com/gifs/no-tina-fey-inside-the-actors-studio-EQTwY2NNikooM

Although this is what we did in the activity in class. In general, this is NOT what we will do. There are two reasons.

  1. The test set error rate could vary quite a bit depending on the particular test set that was randomly held out.

AND, more importantly,

  1. We want to reserve this test set to compare our final few models, likely after we have tried numerous algorithms to predict our response.

Let’s look at the big picture of our build/train –> evaluate loop again.

Build/train –> Evaluate loop

Image credit: https://bradleyboehmke.github.io/HOML/process.html

Just to overemphasize this point, notice again the testing data is left out until the very end. That is what we will use to compare the few best models we have after trying different algorithms.

In order try to find the best model within a specific algorithm, for example the best linear regression model, we only want to use the training dataset.

This process of finding the best model within an algorithm is often called tuning the model. In linear regression, usually what we are tuning is the number of variables used in our model. In other algorithms there will be other parameters that need to be tuned.

So, how do we honestly assess how well our model performs on data it hasn’t seen when we are only using the training data? We sample from the training data!

Within each of these samples, a piece is reserved for fitting the model (represented by the blue Analyze boxes in the schematic) and another piece for assessing the performance of the model (represented by the green Eval boxes in the schematic). This process is repeated many times so we can average the performance over different samples.

There are many ways we can do the sampling. Today we will talk about a technique called k-fold cross-validation. Or k-fold CV, for short.

\(K\)-fold Cross-Validation

Image credit: https://bradleyboehmke.github.io/HOML/process.html#resampling

In \(k\)-fold cross-validation, we start by dividing the data randomly into \(k\) approximately equal groups or folds. The schematic here shows 5-fold cross-validation.

The model is fit on \(k-1\) of the folds and the remaining fold is used to evaluate the model. Let’s look at the first row in the schematic. Here the model would be fit on the data that are in folds 2, 3, 4, and 5. The model would be evaluated on the data in fold 1. If we are fitting a linear regression model, that means we would compute the RMSE for fold 1. In the second row, the model is fit on the data in folds 1, 3, 4, and 5 and the RMSE would be computed for the data in the 2nd fold.

After this is done for all 5 folds, we take the average of the performance statistic, like RMSE for linear regression models, to obtain the overall performance. This overall error is sometimes called the CV error. Averaging the performance over \(k\) folds gives a better estimate of the true error than just using one hold-out set. It also allows us to estimate its variability.

Although it somewhat depends on the size of the dataset, 5 and 10 fold cross-validation are quite common.

Next, I will briefly introduce two functions from the caret package that will help us do k-fold cross-validation and some other sampling methods later in the course. We will explore these functions more deeply in class.

caret is an acronym for classification and regression training. The functions in caret can be used to perform many different machine learning algorithms without changing too much of the code. We’ll be very thankful for this as we continue to learn more algorithms.

The trainControl() function sets up the sampling methodology before the full cross-validation process is executed. For now, we will only use two of its arguments: method is the sampling method we want to use (“cv” for cross-validation now, but we will use others later) and number is the number of folds in cross-validation.

Once that is set up, we can move on to the train function where the cross-validation is actually executed. Since random sampling will be used, we must first set a seed so we can reproduce our results.

The first argument in the train function is the model formula, just like what you’re used to using in lm. This can include interaction terms and other transformations if you’d like.

The data argument will either being the training data after the data have been split into training and test datasets, or it could be a “baked” training dataset after applying some transformations using functions from recipes. The method is the model fitting method or machine learning algorithm. For now, we’ll use lm. The trControl argument is the sampling methodology. It needs to be the result of the trainControl function. na.action = na.omit will omit any row with missing values. Be careful with this if there are a lot of missing values in the dataset.

First, split the data into training (70%) and test (30%):

cars2018 <- read_csv("https://raw.githubusercontent.com/juliasilge/supervised-ML-case-studies-course/master/data/cars2018.csv")
set.seed(345)
cars_split <- initial_split(cars2018, prop = .7)
cars_train <- training(cars_split)
cars_test <- testing(cars_split)

Perform 5-fold CV on the cars_train data, using all variables except Model and Model Index:

splits <- trainControl(method = "cv", number = 5)

set.seed(100)
cars_cv <- train(
  MPG ~ .,
  data = cars_train %>% select(-Model, -`Model Index`), 
  method = "lm",
  trControl = splits,
  na.action = na.omit
)

We will use the data on cars from the EPA as an example. First, we split the data into training data and test data, putting 70% in the training dataset.
