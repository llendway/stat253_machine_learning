Review: model notation

Goal: model response \(y\) using predictors \((x_1, ..., x_p)\)

\[ y = f(x) = f(x_1, x_2, ..., x_p) + \varepsilon \space \space (1) \]

Linear regression:

\[ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + ... + \beta_p x_p + \varepsilon \space \space (2) \]

Each observation can be written as: \[ y_i = \beta_0 + \beta_1 x_{i1} + \beta_2 x_{i2} + ... + \beta_p x_{ip} + \varepsilon_i \space \space (3) \]

The estimated model is:

\[ \hat{y} = \hat{\beta}_0 + \hat{\beta}_1 x_1 + \hat{\beta}_2 x_2 + ... + \hat{\beta}_p x_p, \space \space (4) \]

where \(\hat{\beta}_j\)s are the values of \(\beta_j\) that minimize the sum of the squared residuals:

\[ RSS = \sum_{i = 1}^n \Big[y_i - \big(\beta_0 + \beta_1 x_{i1} + ... + \beta_p x_{ip} \big) \Big]^2 = \sum_{i = 1}^n \Big(y_i - \beta_0 - \sum_{j=1}^p \beta_j x_{ij} \Big)^2 \space \space (5) \]

Lets take a bit of time to review some notation from linear regression. We will build on this notation in this course.

In the regression setting, we have a quantitative response variable \(y\). The response variable is also called the target, outcome, dependent variable, and variable of interest. Our goal is to model the trend in \(y\) using a function of the \(p\) explanatory variables. Explanatory variables are also called predictors, features, and independent variables. Linear models make the special assumption that the function \(f(x)\) is a linear combination of the explanatory variables, as shown in equation 2.

Each individual observation can be written as in the 3rd equation. Then, the estimated model (also called fitted line, model formula, model, line of best fit, least squares line), shown in the 4th equation. The \(\hat{\beta}_j\)s are the values of \(\beta_j\) that minimize the sum of the squared residuals, shown in the 5th equation).

Fitted values, \(\hat{y}_i\) are found by plugging in the values of the \(i\)th observations predictor variables into the model. The residual is the difference between the actual value, \(y_i\) and the fitted value \(\hat{y}_i\).

The R model output, \(R^2=\) 0.55 and \(\hat{\sigma}=\) 3.75.

These data show the engine displacement vs.the miles per gallon (MPG) for 2018 vehicles originally obtained from the EPA and further cleaned and made publicly available by Julia Silge.

We can see from the graph and the model output that there is a negative, somewhat linear, moderate relationship. The blue line shows the fitted line. The purple vertical line highlights engine displacement values of 3 cubic inches. The purple horizontal line reflects the fitted value, \(\hat{y}_i\) for an observation with a displacement of 3. The yellow vertical line highlights engine displacement values of 4 cubic inches. The yellow horizontal line reflects the fitted value, \(\hat{y}_i\) for an observation with a displacement of 4. The distance between the purple and yellow lines is about -3.13, which reflects the estimated slope. On average, cars with a displacement that is 1 cubic inch larger will get 3.13 MPG less. The intercept, 32.9, is interpreted as the average MPG for a car with a displacement of 0 cubic inches.

The red line segments represent residuals - the distance between the actual MPG and the fitted MPG and are highlighted for just a handful of observations.

The model \(R^2\) tells us that about 0.55 of the variation in MPG is explained by engine displacement. So theres still quite a bit of unexplained variation. The \(\hat{\sigma}\) means that, on average, predicted MPGs will be off of their actual MPGs by about 3.75 MPG.

Overview of ML process

In order to better understand where linear regression fits into the machine learning process, lets look at our course flow diagram. If you rewind to the beginning of the video, you will notice that this is an expansion of what was shown in the first graphic. That is because linear regression is a type of machine learning!

In your 155 class, you likely focused on interpretation of the model, with a small amount of time dedicated to how you might use it for prediction. In this class, we will focus more on how to use the model for prediction, and we will be very interested in finding models that have the best predictive value. That being said, we still think its nice when we can understand the model. So, we wont completely drop the interpretation piece.

Next, lets examine a bit closer what happens inside the build/train > evaluate loop.

The modeling process: inside the build / train > evaluate loop

This is where we see a big difference from what we are used to in STAT 155. Think of fitting one linear regression model as one of the analyze> eval pairs.

In this class, in order to better evaluate the performance of our models, we will divide up the data, fit models on a portion of the data, and evaluate how well it performs on the portion that wasnt used to fit the model. We will do this on different samples of data and we also might try different types of models or algorithms. This will help us tune the parameters of whichever algorithm we are using.

We also hold out a special portion of the data that we refer to as the test data. We save this portion until the very end when we want to evaluate the best candidate models to make a final decision about which one we will use.

I have simplified this for now, but the details will come as we work through the course.
