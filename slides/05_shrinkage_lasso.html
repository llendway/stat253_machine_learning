Review of linear models

The estimated coefficients, \(\hat{\beta_1}, \hat{\beta_2}..., \hat{\beta_p}\), are those that minimize the residual sum of squares:

\[ RSS = \sum_{i = 1}^n \Big[y_i - \big(\beta_0 + \beta_1 x_{i1} + ... + \beta_p x_{ip} \big) \Big]^2 = \sum_{i = 1}^n \Big(y_i - \beta_0 - \sum_{j=1}^p \beta_j x_{ij} \Big)^2 \]

When we fit linear models using ordinary least squares, the estimated coefficients, \(\hat{\beta_1}, \hat{\beta_2}..., \hat{\beta_p}\) , are those that minimize the residual sum of squares, RSS.

That is, we find the \(\beta_1, \beta_2..., \beta_p\) such that the sum of the squared differences between the actual and fitted values is as small as possible.

Keep in mind, RSS will decrese as the number of terms in our model increases, even if those terms are random noise, or garbage variables as I like to all them.

LASSO

The Least Absolute Shrinkage and Selection Operator finds \(\hat{\beta_1}, \hat{\beta_2}..., \hat{\beta_p}\) by minimizing:

\[ \begin{align} \sum_{i = 1}^n \Big(y_i - \beta_0 - \sum_{j=1}^p \beta_j x_{ij} \Big)^2 & + \lambda \sum_{j=1}^p |\beta_j|, \text{ where } \lambda\ge 0 \\ \text{RSS} \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ & + \ \ \text{penalty} \end{align} \]

Competing criteria:

  • RSS is small when coefficients fit data (ie. the more variables the better).
  • Penalty is small when coefficients are small - it is smallest when they are all 0!

The LASSO, which stands for least absolute shrinkage and selection operator minimizes a different quantity than in least squares regression. But, notice that the first part is still the residual sum of squares. The second part is what is referred to as a penalty term and it is the sum of the absolute value of the coefficients times a parameter \(\lambda\).

These two pieces compete with each other. The RSS piece on the left will be smaller when more terms are in the model and coefficient estimates fit the data well.

The penalty term on the right will be smaller when coefficients are small. It is smallest when the coefficients are all equal to zero. So the penalty term will shrink coefficients, sometimes even shrinking them to zero.

Tuning

\[ \begin{align} \sum_{i = 1}^n \Big(y_i - \beta_0 - \sum_{j=1}^p \beta_j x_{ij} \Big)^2 & + \lambda \sum_{j=1}^p |\beta_j|, \text{ where } \lambda\ge 0 \\ \text{RSS} \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ & + \ \ \text{penalty} \end{align} \]

The extremes: when \(\lambda = 0\) - OLS estimates and all coefficients are included, when \(\lambda = \infty\) - all coefficients are 0 and only an intercept.

Ideal: choose \(\lambda\) so some of the coefficients will shrink to zero. How? Use cross-validation!

Scale variables: make all variables have mean 0 and standard deviation 1 so they are penalized “fairly”.

The \(\lambda\) tuning parameter allows us to choose how much to penalize the coefficients. We have two extreme cases. When \(\lambda\) is zero, there is no penalty, and all coefficients are included with regular least squares coefficient estimates. When \(\lambda = \infty\), the coefficients would would all be zero and we end up with a model with only an interecept.

So, as \(\lambda\) gets larger, some of the coefficients will shrink to zero. These are typically the weaker predictors. Since some of the coefficients are shrinking to zero, this method is a form of model selection. Only the terms with non-zero coefficients are used in the model. We will use cross validation to choose the best \(\lambda\).

And, one more important piece before moving on to a couple examples. Because we are penalizing coefficients, it is important that we put our variables on the same scale so that coefficients are being penalized fairly. In the functions we use, they will be automatically scaled to have mean zero and standard deviation one. If we didn’t do this, variables that are measured on larger scales would be penalized more than those measured on smaller scales. For example, the same variable measured in pounds would be penalized more than if it were measured in ounces. Again, the functions we use will do this part for us. It will also display the final coefficients in the original scale so that the model is interpretable.

More complex example - simulated credit data

The data used in this example are from the ISLR package that goes with the Introduction to Statistical Learning textbook. There are 400 simulated observations of credit card data. The response variable is Balance, which is the average credit card balance in $. There are more than 10 explanatory variables, but we focus on Income, credit rating, and a Student indicator variable.

Both graphs on the left are like the graph in the lower right on the previous slide. They show the estimated coefficients from the lasso for various values of \(\lambda\). The one on the bottom is zoomed into a smaller range of values on the y-axis to show the details more clearly, but in doing that, we lose view of the Student variable. In general, we see that as \(\lambda\) increases, more variables have estimated coefficients of zero.

How do we choose the “best model.” We use cross-validation, of course! The graph on the right shows the cross-validated rmse for various values of \(\lambda\). Lines are extended 1 standard error in each direction from each cross-validated standard error. The pink vertical indicates the value of \(\lambda\) with the smallest cross-validated rmse. We could choose this as our best model.

But, if we really want a simple model and we know that there is some variation in the rmse, it could also be a good idea to choose a simpler model with an rmse that’s still close to the smallest rmse. What is close? It is recommended to choose the largest value of \(\lambda\) such that its cross-validated rmse is still within one standard error of the smallest cross-validated rmse. The turquoise vertical line indicates that \(\lambda\). The pink and turquoise lines are also represented in the graphs on the left so you can get a sense of the difference between those models.

In the graph on the right, the red horizontal line is the value of the smallest cross-validated rmse plus one standard error. So, we are looking for the largest \(\lambda\) whose rmse is below that red line. A larger \(\lambda\) will give us a simpler model. So, as mentioned before, we often choose this value of \(\lambda\) rather than the \(\lambda\) with the smallest rmse.
