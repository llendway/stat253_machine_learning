Where are we?

This video introduces decision trees, specifically regression trees, which will nearly bring us to the end of regression methods for this class.

Decision trees are a non-parametric method, sometimes referred to as a rules-based model

Main Idea

Decision trees are really just a complicated list of if-else rules that bin observations together. Observations in the same bin, also called nodes or leaves, have the same predicted value. In the regression trees we will discuss the prediced value will be the average of the response variable for that node.

One of my favorite aspects of this modeling method is the ease with which you can explain it to other people, including people with very little statistical training. This is largely due to being able to display the results in a nice graphic.

Lets take a look at the simple regression tree we have here. This is using the MPG data we have seen before. In the top node, we see some details about the entire training dataset. The 100/% means that 100/% of the data is contained in that node. The average MPG for the entire training dataset is 23 MPG as indicated by the 23.

Beneath the box, we see the first split: Displacement >= 2.6. Observations that have a displacement greater than 2.6 follow the “yes” branch, to the left in this diagram. Those that dont follow the “no” branch to the right.

Main Idea

Focusing on the yes branch, we see that 55% of the data end up in that node and the average MPG is 20 MPG. This node is then further split on displacement being greater than 3.9. If it is, the observation falls into the node on the left where 22% of the data reside and the average MPG is 17.

Observations with displacement less than 3.9 (but still greater than 2.6 from the initial split) fall into the second node where 33% of the data reside and the average MPG is 21.

The tree continues splitting until some stopping criteria is satisfied.

Main Idea

To use a regression tree for prediction a new observation would follow the path until reaching a terminal node, which is a node that has no branches coming out of it. That nodes average response would be its predicted value.

So, for example, a car with a displacement less than 1.9 would follow the path indicated in red and would have a predicted MPG of 32 MPG.

Splitting rules

In regression trees, the splits are decided by choosing the variable and value of that variable that minimize the total sum of squared residuals after the split. Mathemetically, define

\[ R_1(j,s) = \{X | X_j < s\} \ \ \text{and} \ \ R_2(j,s) = \{X | X_j \ge s\} \] We want to find \(X_j\) (the variable) and \(s\) (the value of that varible) that minimize

\[ SSE = \sum_{i: x_i \in R_1(j,s)}(y_i - \bar{y}_{R_1})^2 + \sum_{i: x_i \in R_2(j,s)}(y_i - \bar{y}_{R_2})^2 \]


Im sure by now you are wondering how we decide how to make the best splits in the tree. We have two goals: 1. Make the observations within each node as similar as possible, and 2. Make the nodes different from one another.

We will do this by finding the split that minimizes the sum of the squared residuals, shown in the bottom equation.

Relating our example to this equation, lets look at the first split. Here, \(X_j\) is displacement, \(s=2.6\), \(R_2\) is the node with the observations with displacement greater than or equal to 2.6 and \(R_1\) is the other node.

The SSE can be computed by subtracting the average MPG for the observations in \(R_2\), 20 MPG, from each observation in \(R_2\), squaring that quantity and summing over all observations. Do the same for \(R_1\) where the averge MPG is 27 MPG and add them both up.

Any other split with any other variable would have led to a larger SSE.

This splitting process would keep going until a reasonable stopping criteria is reached.

Stopping

Speaking of stopping criteria. How do we decide to stop? First, why would we want to stop?

Like the tree we see here, if we dont stop, we will end up with nodes with very few data points in them. What do you think would happen if we tried to apply this to a new dataset? (WAIT)

Yes, thats right, it will perform poorly on the new data because we overfit the model to the training data.

There are many methods to decide when to stop. We will use a parameter generically called cp, which stands for complexity parameter. In regression trees, the overall R-squared must increase by at least cp at each split. If it does not increase by at least that much, it will not create the split. We can test different values of cp in our usual way, using cross-validation.

Implementing in R

set.seed(327)
mpg_tree_cp <- train(
  MPG ~ Displacement + Cylinders + Gears + 
    Transmission + Aspiration + Drive,
  data = cars_train, 
  method = "rpart",
  trControl = trainControl(method = "cv",
                           number = 5),
  tuneGrid = data.frame(cp = 10^seq(-4, -2 , 
                                    length = 10))
)
